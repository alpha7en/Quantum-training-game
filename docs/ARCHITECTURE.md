# Архитектура Quantum Training Game

## 1) Движок (Engine)

Ключевой модуль: `src/quantumMath.ts`.

- Комплексные числа и операции (`cAdd`, `cMul`, `cConj`, `cNorm`).
- Матрицы/векторы, тензорные произведения.
- Унитарная эволюция:
  - для вектора состояния `|ψ'⟩ = U|ψ⟩`;
  - для матрицы плотности `ρ' = UρU†`.
- Построение управляемых операций (включая несколько control-точек в одном столбце).
- Проверка унитарности и вычисление fidelity:
  - state fidelity;
  - unitary fidelity (с точностью до глобальной фазы).
- Парсер математических выражений углов: `pi`, `pi/2`, `2*pi/3`, скобки, знаки, `+ - * /`.

## 2) UI (Конструктор и визуализация)

Ключевой модуль: `src/CircuitBuilder.tsx`.

- Схема хранится как таблица `numQubits × numColumns`.
- Парадигма: **1 столбец = 1 такт**.
- Drag-and-drop реализован через pointer events (под touch и desktop).
- Snap-to-grid обеспечен вычислением целевой ячейки по координате pointer.
- Контроли:
  - гейт `C` как управляющая точка;
  - автоматическая SVG-линия между control и target в столбце.
- Инициализация кубитов:
  - пресеты `|0⟩, |1⟩, |+⟩, |−⟩`;
  - custom pure через параметры на сфере Блоха (`θ, φ`);
  - custom mixed как `p0|0⟩⟨0| + p1|1⟩⟨1|`.
- Параметризованные гейты (`Rx/Ry/Rz/P`) редактируются в Bottom Sheet (`src/GateConfig.tsx`).
- Справка по гейтам (`src/GateHelp.tsx`) с KaTeX-рендерингом формул.

## 3) Задачи и верификация

Ключевой модуль: `src/TaskManager.tsx`.

- Два режима:
  - экзаменационные билеты (`ticketsData`);
  - игровые задачи (`challengeTasks`).
- Ограничения задач:
  - разрешённые гейты;
  - фиксированные гейты;
  - lock initial states / lock qubits;
  - разрешённые параметры.
- Проверка решений:
  - target state: fidelity с конечной матрицей плотности;
  - target unitary: unitary fidelity.
- Выравнивание глобальной фазы используется для удобного сравнения отображаемых амплитуд/матриц.
- Shots-режим:
  - построение распределения результатов измерения после заданного числа запусков;
  - поддержка измерения в базисах Z/X/Y через соответствующее базисное преобразование.

## 4) PWA и оффлайн

- `public/manifest.json` — standalone приложение для мобильного запуска.
- `public/sw.js`:
  - pre-cache базовых ресурсов;
  - stale-while-revalidate для cached assets;
  - fallback на `index.html` при offline navigation.
- Регистрация service worker в `src/main.tsx` в production.
- `vite.config.ts` задаёт `base` для GitHub Pages через `GITHUB_REPOSITORY`.
