# Соответствие исходному ТЗ: статус

Ниже — честная фиксация текущего состояния проекта относительно большого ТЗ.

## Реализовано

- Веб-приложение без backend, вся логика в браузере.
- PWA-основа: `manifest.json`, service worker, оффлайн-кэш.
- Конструктор схем с grid-парадигмой «1 столбец = 1 такт».
- Touch-friendly drag-and-drop через pointer events.
- Набор ключевых гейтов: H, X, Y, Z, Rx, Ry, Rz, P, controls, Mx/My/Mz.
- Несколько control-точек + автолиния между control и target.
- Bottom Sheet для настройки параметризованных гейтов.
- Парсер угловых выражений (`pi/2`, `-pi/4`, `2*pi/3`).
- Справка по гейтам с KaTeX (матрицы/формулы/действия на базис).
- Инициализация кубитов: базовые состояния + custom pure + custom mixed.
- Результаты симуляции: state vector (если применимо), density matrix, вероятности, Bloch.
- Shots-режим с гистограммой статистического распределения.
- Задачи с ограничениями (фиксированные гейты, whitelist гейтов и параметров).
- Автопроверка задач по fidelity (с учётом глобальной фазы при сравнении представления).

## Частично реализовано / упрощено

- Смешанные состояния на входе реализованы как смесь по каждому кубиту вида
  `p0|0⟩⟨0| + p1|1⟩⟨1|` (без общего UI для произвольной полной ρ на всю систему).
- Ввод произвольных амплитуд в формате `α|0⟩ + β|1⟩` напрямую текстом не выделен отдельным режимом;
  вместо этого используется параметризация через углы Bloch (`θ`, `φ`).
- Функция «долгий тап (long press)» как отдельный жест не выделена явно;
  доступ к справке реализован через `ⓘ`/контекстное меню.

## В roadmap (рекомендуется)

- Полный редактор матрицы плотности для всей системы (или для отдельных подсистем).
- Явный режим long-press для мобильной UX-навигации.
- Более строгий и расширяемый парсер пользовательских ответов (векторы/матрицы из текста).
- Набор автотестов для touch-сценариев (`touchstart/touchmove/touchend`) и регрессий DnD.
- Профилирование производительности (5–8 кубитов и выше) с отчётными метриками.
- Формализованный документ архитектурных решений (ADR) и расширенная техдокументация API.
